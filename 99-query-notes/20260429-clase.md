# Notas de clase 20260429

Comandos para que SQLite despliegue las consultas en tablas con los nombres de
las columnas y la palabra `Nulo` en donde el valor sea `NULL`

```
.mode box
.headers on
.nullvalue Nulo
```

Revisamos por qué falló la carga de archivos `csv` cuando se tenía la restricción 
de `sex`:

```sql
SELECT DISTINCT sex FROM Surveys;
SELECT DISTINCT sex FROM Surveys WHERE sex = ""; -- Mala idea
SELECT DISTINCT sex FROM Surveys WHERE sex IS NULL;
```

Actualizar los valores de `sex` como `NULL` si no están en los valores permitidos
`M` o `F`

```sql
UPDATE 
    Surveys
SET 
    sex = NULL
WHERE
    sex NOT IN ("M", "F");
```

> [!CAUTION]
> Este cambio es una mala idea cuando se trabaja en una base de datos que no se
> sabe si está bien codificada ya que pueden existir datos codificados como `Male`
> los cuales al aplicar este cambio serán remplazados por el valor `NULL` 
> perdiendo así información

Se actualiza los valores de la columna `hfl` de la tabla `Surveys` a `NULL` si 
son cadenas vacías.

```sql
UPDATE 
    Surveys
SET 
    hfl = NULL
WHERE
    hfl = "";
```

Esta consulta tiene como proposito verificar que las mediciones sean menores que 
el mayor registrado para la especie:

```sql
SELECT 
    sps.speciescode,
    sps.maxhfl,
    occ.species, 
    occ.hfl 
FROM 
    Surveys AS occ, 
    Species AS sps 
WHERE 
    sps.speciescode=occ.species AND 
    occ.hfl IS NOT NULL AND 
    occ.hfl > sps.maxhfl;
```
